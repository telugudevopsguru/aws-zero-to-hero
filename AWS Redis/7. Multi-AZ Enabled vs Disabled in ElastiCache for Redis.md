## ‚úÖ **Multi-AZ Enabled vs Disabled in ElastiCache for Redis**

| Feature                    | **Without Multi-AZ**                                      | **With Multi-AZ**                                                   |
| -------------------------- | --------------------------------------------------------- | ------------------------------------------------------------------- |
| **Architecture**           | All nodes in **a single AZ**                              | Primary in one AZ, **replicas in other AZs** (e.g., AZ-A, AZ-B)     |
| **High Availability (HA)** | ‚ùå Not highly available ‚Äî Single point of failure          | ‚úÖ Designed for **HA** ‚Äî automatic failover to replica in another AZ |
| **Automatic Failover**     | ‚ùå No automatic failover                                   | ‚úÖ Yes ‚Äî automatic failover to the replica on primary failure        |
| **Disaster Recovery**      | ‚ùå Limited ‚Äî all nodes lost if AZ fails                    | ‚úÖ Redis instance remains available even during **AZ outage**        |
| **Read Replica Placement** | All replicas in same AZ                                   | Replicas spread across **multiple AZs**                             |
| **Write Availability**     | Downtime if the node fails                                | **Minimal** downtime ‚Äî quick failover to healthy replica            |
| **Reader Endpoint Usage**  | Can be used, but not across AZs                           | ‚úÖ Reader endpoint distributes reads across **multi-AZ replicas**    |
| **Maintenance**            | May cause brief downtime                                  | ‚úÖ Failover reduces impact during maintenance                        |
| **Cost**                   | üí∞ Lower (fewer cross-AZ traffic & fewer replicas needed) | üí∞ Slightly higher (extra replicas and inter-AZ data transfer)      |
| **Use Case**               | Non-critical, dev/test workloads                          | Production, **mission-critical applications**                       |


## üß† Notes:

* Multi-AZ is supported **only with Redis Cluster Mode Disabled (single shard)** and **enabled with replicas**.
* For best HA, always use **Multi-AZ + Automatic Failover + Redis Auth**.
* Use **reader endpoints** to offload reads from the primary.
* If performance is key, consider **adding more replicas** or **using cluster mode**.
